方法论与数据说明

Token 产出估算：日 Token 产出 = 年化收入 ÷ 加权均价 ÷ 365。加权均价假设 80% 流量来自经济/中端模型、20% 来自旗舰模型，输出 Token 占比 70%。如公司有官方 Token 产出数据（如字节跳动日均 1200 万亿 tokens），则直接采用。

收入结构三分法：消费者订阅（ChatGPT Plus、Claude Pro 等）、API / 开发者（API 调用，含 Cursor、Claude Code 等间接贡献）、企业与合作伙伴（企业部署、Azure OpenAI 等大客户合同）。

数据可信度分三级： 官方披露（财报/SEC 文件） 媒体报道（The Information 等） 分析师估算（需注明假设）

企业梯队划分：第一梯队为有官方财报或可靠收入数据的头部企业（Google、OpenAI、Anthropic、Microsoft 等）；第二梯队为依赖媒体报道的企业（xAI、Mistral AI 等）；第三梯队为数据有限、以估算为主的企业（百度、阿里巴巴等）。

数据来源

  • 上市公司 SEC 10-K / 10-Q 财报及季度财报电话会议
  • CEO / CFO / 高管公开声明与媒体采访
  • The Information, Bloomberg, CNBC, Financial Times 等科技媒体报道
  • 各公司官方 API 定价页面及开发者文档
  • SimilarWeb、Sensor Tower 等第三方流量分析平台

免责声明

非上市公司收入数据多为媒体报道或分析师估算，可能存在较大误差。Token 消耗量为基于均价的反推估算，非官方数据。数据置信度通过颜色标识（🟢 官方 / 🟡 报道 / 🟠 估算）。本项目仅供研究参考，不构成投资建议。
